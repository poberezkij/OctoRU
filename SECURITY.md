# Security Policy

## Supported Versions

Security fixes are provided for the latest release branch (`main`) and the latest published version.

## Reporting a Vulnerability

If you found a security issue, please do not publish it in public issues.

Please report privately to the maintainer:
- GitHub: https://github.com/poberezkij

When reporting, include:
- Steps to reproduce
- Affected pages/features
- Potential impact
- Suggested fix (optional)

Response target:
- Initial response: within 7 days
- Fix timeline: depends on severity and complexity

---

# Политика безопасности

## Поддерживаемые версии

Исправления безопасности выпускаются для актуальной релизной ветки (`main`) и последней опубликованной версии.

## Сообщение об уязвимости

Если вы нашли уязвимость, пожалуйста, не публикуйте её в открытых Issues.

Сообщите о проблеме приватно мейнтейнеру:
- GitHub: https://github.com/poberezkij

В сообщении укажите:
- шаги воспроизведения;
- затронутые страницы/функции;
- потенциальное влияние;
- возможное исправление (необязательно).

Ориентиры по срокам ответа:
- первичный ответ: до 7 дней;
- срок исправления: зависит от критичности и сложности.
